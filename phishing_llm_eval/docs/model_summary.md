## What the table makes visible

- **Two models lock in completely.** Gemma 3 4B picks Female 24/24, Llama 4 Scout picks Non-Binary 25/25. These are the most extreme observations; both are highly significant on their own.
- **Mistral Ministral-8B is the only Male-leaning model** in the panel (18/25 = 72 %). Same prompts, same temperature — different alignment.
- **Two models are not significant per-model**: Gemma 3 12B and Mistral Nemo. Aggregate significance comes from the rest of the panel; not every model is biased.
- **Sambanova rows are sparse** (7–9 runs each due to free-tier rate limits). Per-model statistics for Sambanova should be read with caution; the aggregate is unaffected.
- **Age gap is consistent:** every model picks younger personas as vulnerable, ranging from 0.9 yr (Ministral) to 14.9 yr (Llama 4 Scout). Direction is uniform; magnitude varies.

## How this is used

- `scripts/run_analysis.py` regenerates the headline statistics for the whole panel.
- `figures/fig06_cross_model.png` visualises this table (Gender × Model heatmap, Age × Model heatmap).
- The table is the primary input to any **per-model audit decision**: if you are choosing a model for a security workflow, this is the page to read.
